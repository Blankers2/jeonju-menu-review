# 메뉴판 추출·검수 도구 설계 (Menu Finder)

작성일: 2026-06-08
작성자: jaeman.lee@flitto.com (with Claude)

## 1. 목표 (Goal)

약 400장의 식당 메뉴판 원본 이미지에서 **메뉴명·가격**을 가게별로 추출한다.
OCR이 완벽하지 않으므로 **사람이 보고 빠르게 수정·확정**하는 것을 전제로,
"처음부터 타이핑"이 아니라 "초안을 보고 고치기"가 되도록 작업을 효율화한다.

최종 산출물: 가게별 CSV `place_id, 가게명, 메뉴명, 가격` (+ 전체 합본 CSV).

## 2. 핵심 결정 사항 (Decisions)

| 항목 | 결정 | 근거 |
|------|------|------|
| 입력 이미지 | **원본만 사용** (인페인팅 미사용) | 실제 작업 폴더엔 원본만 존재. OCR이 자체 검출하므로 인페인팅 diff 불필요. 필요시 후속 보강 가능(아래 6장). |
| OCR 엔진 | **PaddleOCR(한국어) 기본 + 교체 가능 인터페이스** | 무료. 외부전송 허용되므로 추후 Google Vision 등 더 강한 엔진으로 한 줄 교체 가능하게 추상화. |
| 외부 전송 | **허용** | 단순 텍스트 추출이라 민감정보 아님. 엔진 선택지 확대. |
| 산출 형식 | 가게별 CSV `place_id, 가게명, 메뉴명, 가격` + 합본 CSV | 범용·후가공 용이. |
| 소/중/대 다중가격 | **사이즈마다 별도 행** | 예: `장어탕(소),10000` / `장어탕(중),13000`. 한 행=한 가격 원칙. |
| 이미지 로드 | **드래그앤드롭** (이미지/폴더째) | 고정 폴더 파이프라인 제거, 앱 하나로 통합. |
| 가게명 | 파일명에서 자동 추출 후 화면 표시, **작업자가 수정 가능** | |
| PlaceID | 추출 가게명을 PlaceID 목록과 **자동 매칭(후보 제시)** → 작업자 확정 | PlaceID가 PK. 매핑표는 `data/place_ids.csv`. |

## 3. 전체 구조 (Architecture)

단일 로컬 웹앱. 별도 배치 스크립트 없이 드래그앤드롭으로 인입한다.

```
┌─────────────────────────── 브라우저 (검수 UI) ───────────────────────────┐
│  드래그앤드롭 영역  →  업로드                                             │
│  좌: 원본 이미지 + OCR 박스 오버레이(토글)   |   우: 편집 표 [메뉴명|가격] │
│  가게 사이드바 + 진행률(검수완료 N/총)        |   가게명/PlaceID 헤더(수정)│
│  CSV 내보내기 버튼                                                        │
└───────────────────────────────┬──────────────────────────────────────────┘
                                 │ HTTP(JSON) + 이미지 정적 서빙
┌────────────────────────────────▼──────────────────────────────────────────┐
│  FastAPI 백엔드                                                             │
│   - 업로드 수신 → 디스크 저장(images/)                                      │
│   - 백그라운드 OCR 큐(워커) → 진행률 폴링 API                               │
│   - OcrEngine 인터페이스(기본: PaddleOcrEngine)                             │
│   - 자동 매칭기(MenuPairer): 박스 → 컬럼/행 클러스터링 → 메뉴↔가격 쌍       │
│   - 가게명 추출 + PlaceID 퍼지매칭(StoreResolver)                           │
│   - 초안/수정본 영속화(store JSON)  ·  CSV 익스포터                         │
└─────────────────────────────────────────────────────────────────────────┘
```

데이터는 디스크에 캐시: 한 번 OCR한 이미지는 재방문해도 재OCR하지 않는다.

## 4. 구성 요소 (Components)

각 모듈은 단일 책임 + 명확한 인터페이스를 갖는다.

### 4.1 OcrEngine (인터페이스)
- 책임: 이미지 1장 → 텍스트 박스 리스트.
- 입력: 이미지 경로/바이트. 출력: `[{text, bbox:[x,y,w,h], confidence, polygon}]` (좌표는 원본 픽셀 기준).
- 기본 구현 `PaddleOcrEngine`(lang=korean). 향후 `EasyOcrEngine`, `VisionApiEngine` 추가 가능.
- 의존: PaddleOCR. 교체 시 이 인터페이스만 만족하면 됨.

### 4.2 MenuPairer (자동 매칭기)
- 책임: OCR 박스 리스트 → 초안 메뉴 행 `[{menu_name, price, source_boxes:[...]}]`.
- 로직:
  1. 박스를 x좌표로 **컬럼 클러스터링**(좌/우 페이지, 다단 메뉴 대응).
  2. 컬럼 내에서 y좌표 기준 **행 클러스터링**(기울기 허용: 인접 박스 각도 추정).
  3. 각 박스를 **가격/메뉴명 분류**: 숫자+`,`+선택적 `원`/통화 패턴이면 가격, 아니면 메뉴명.
  4. 같은 행 밴드에서 메뉴명(좌)–가격(우) 페어링. 한 행에 가격 여러 개면 메뉴명 1개에 대해 **가격 수만큼 행 생성**(소/중/대).
- 출력은 어디까지나 **초안**. 틀려도 검수 UI에서 사람이 교정.
- 의존: 없음(순수 기하 로직). 단위 테스트 용이.

### 4.3 StoreResolver (가게명·PlaceID)
- 책임: 파일명 → 추출 가게명 + PlaceID 후보.
- 파일명 정규화: 끝의 일련번호, `-crop`, `_merged`, `(보완)`, 확장자 제거.
- 같은 정규화 이름 = 같은 가게로 그룹핑(여러 장 → 한 가게).
- `data/place_ids.csv`와 **퍼지매칭**(부분일치/유사도)으로 PlaceID 후보 top-N 제시. 확신 시 자동 선택, 애매하면 작업자 선택.
- 의존: place_ids.csv.

### 4.4 Store / DraftStore (영속화)
- 책임: 가게별 작업 상태 저장/로드. JSON 파일 1개/가게.
- 스키마: `{place_id, title_extracted, title_confirmed, images:[{filename, ocr_boxes, rows:[{menu, price}], reviewed:bool}], status}`.
- 자동 저장(편집 시 디바운스).

### 4.5 CsvExporter
- 책임: 확정 데이터 → 가게별 CSV + 전체 합본 CSV. UTF-8 (Excel 호환: BOM 또는 UTF-8-sig).
- 컬럼: `place_id, 가게명, 메뉴명, 가격`.

### 4.6 Web API (FastAPI)
- `POST /upload` (다중 파일) → 저장 + OCR 큐 등록.
- `GET /progress` → OCR 진행률.
- `GET /stores` → 가게 목록 + 검수 진행률.
- `GET /stores/{id}` → 가게 상세(이미지·박스·행).
- `PUT /stores/{id}` → 수정 저장(가게명/PlaceID/행).
- `POST /reocr/{image}` → 단일 이미지 재OCR(선택).
- `GET /export` / `GET /export/{id}` → CSV.
- 정적: 업로드 이미지 서빙, 프론트(index.html).

### 4.7 Frontend (빌드 불필요한 순수 JS SPA)
- 드래그앤드롭 인입 + 업로드/진행률.
- 좌측 이미지 뷰어: 줌/팬, OCR 박스 오버레이 토글.
- **박스 ↔ 표 행 양방향 클릭 연동**(박스 클릭→행 하이라이트, 행 클릭→박스 하이라이트).
- 우측 편집 표: 인라인 편집, 행 추가/삭제, **소/중/대 행 분할** 버튼, 드래그 재정렬.
- 헤더: 가게명(수정) + PlaceID 자동완성 드롭다운.
- 가게 사이드바 + 진행률, "검수완료" 토글, CSV 내보내기.

## 5. 데이터 흐름 (Data Flow)

1. 작업자가 이미지(들)를 드롭 → `/upload` → `images/`에 저장, OCR 큐 등록.
2. 워커가 순차 OCR → 박스 산출 → MenuPairer로 초안 행 생성 → StoreResolver로 가게/PlaceID 후보 → DraftStore 저장.
3. 작업자가 가게 선택 → 좌측 이미지+박스, 우측 초안 표 확인 → 수정 → 자동 저장.
4. 검수완료 표시 → 진행률 갱신.
5. 전체/가게별 CSV 내보내기.

## 6. 인페인팅 보강 (후속 옵션, 지금은 미구현)

자동 매칭이 좁은/기울어진 메뉴판에서 부정확하면, 인페인팅 이미지를 받아
`원본 − 인페인팅` 마스크로 "지워진 영역=메뉴명/원-붙은 가격, 남은 숫자=가격"을
신호로 활용해 분류·페어링 정확도를 보강한다. OcrEngine/MenuPairer가 교체 가능하므로
별도 모듈로 끼워넣는다. **MVP 범위 밖.**

## 7. 에러 처리 (Error Handling)

- OCR 실패/저신뢰: 박스는 표시하되 표에 빈 칸으로 두고 작업자 입력 유도(빛번짐 영역 등).
- 업로드 중복 파일명: 가게 그룹에 병합(같은 가게 여러 장).
- PlaceID 매칭 실패: 후보 없음 표시, 작업자 수동 선택/입력.
- 잘못 그룹핑된 가게: 작업자가 헤더에서 가게명/PlaceID 재지정.

## 8. 테스트 (Testing)

- MenuPairer: 합성 박스 입력에 대한 단위 테스트(단일컬럼/이단컬럼/다중가격/기울기).
- StoreResolver: 파일명 정규화·퍼지매칭 단위 테스트(예: `궁한정식1-crop`→`궁`? 주의: 부분일치 함정 → 테스트로 검증).
- CsvExporter: 스키마/인코딩 스냅샷 테스트.
- 통합: 케이스모음 이미지 몇 장으로 업로드→OCR→초안 생성 스모크 테스트.

## 9. 기술 스택 (Stack)

- Python 3.x, PaddleOCR(한국어), FastAPI + uvicorn.
- 프론트: 순수 HTML/CSS/JS (빌드 도구 없음).
- 실행(Windows): 의존성 설치 후 `uvicorn app:server` 한 줄 → 브라우저로 `localhost`.

## 10. 범위 밖 (Out of Scope, YAGNI)

- 번역/메뉴 분류/영양정보 등 추출 외 기능.
- 인페인팅 기반 보강(6장, 후속).
- 멀티유저/동시편집/인증.
- 클라우드 배포.
